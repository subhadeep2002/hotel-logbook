Firebase Setup Required
The app is running in local-only mode. Data will reset on page refresh. Follow these steps to connect your Firebase backend.
1
Go to console.firebase.google.com → Create a new project (e.g. royal-stay-inn)
2
In your project → Project Settings → Add App → choose Web (</>) → copy the firebaseConfig object shown
3
Open this HTML file in a text editor → find the section labelled PASTE YOUR FIREBASE CONFIG HERE → replace the placeholder values with your real config
4
In Firebase Console → Firestore Database → Create database → choose Start in test mode (open rules for 30 days)
5
In Firebase Console → Authentication → Sign-in method → Enable Anonymous sign-in
6
For GitHub Pages: push the HTML file to your repo → Settings → Pages → Source: main branch, /root → Save. Your URL will be https://yourusername.github.io/repo-name/hotel-logbook.html
7
Production Firestore rules (replace test rules after go-live):

rules_version = '2';
service cloud.firestore {
  match /databases/{db}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}

The app works fully in local mode — data just won't persist across refreshes until Firebase is configured.
